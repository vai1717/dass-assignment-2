\section*{\textcolor{blue}{Introduction}}
This report documents the black-box testing of the QuickCart REST API. All tests were designed solely from the provided API documentation without any access to the internal implementation. The objective is to verify correct HTTP status codes, proper JSON response structures, and correctness of returned data.

The automated test suite (\texttt{test\_quickcart\_api.py}) contains \textbf{93 test cases} implemented with \texttt{pytest} and \texttt{requests}. Results: 74 passing, 19 bugs found.

% ─────────────────────────────────────────────────────────────────────────────
\section*{\textcolor{blue}{Test Case Design}}

Test cases were designed to cover all five required categories:
\begin{itemize}[noitemsep]
  \item \textbf{Valid requests} — happy-path tests verifying correct 2xx responses and JSON structure
  \item \textbf{Invalid inputs} — malformed or logically invalid data
  \item \textbf{Missing fields} — omitting required headers or body fields
  \item \textbf{Wrong data types} — sending strings where integers are required and vice versa
  \item \textbf{Boundary values} — zero, negative, maximum integers, empty strings, very long strings
\end{itemize}

Each test case below includes: Input (method + URL + headers + body), Expected Output, Actual Output, and Justification.
